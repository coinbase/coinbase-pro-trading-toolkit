Class HFTFilter

Filters the raw WS message stream by batching messages and auto-cancelling zero-effect trades.

If you read from this stream in 'push' mode, i.e. by attaching a data listener to it, then each message will be emitted more/less instantaneously, and so the filtering mechanism won't have time to catch HFTs. The way you probably want to use this stream is to always have it paused and call read when you want the next message (perhaps every n ms). This way, there's maximum opportunity to pre-filter zero-effect trades; and re-order any messages that may have arrived out of order
