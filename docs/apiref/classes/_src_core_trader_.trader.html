Class Trader

The Trader class places orders on your behalf. The commands for placing the trades can either come from an attached stream, or directly via the API.

One should have an authenticated feed piped into Trader so that it can keep track of the state of its own orderbook. Failing this, it is trading 'blind' and will have to rely on REST requests to update the state of the book.

Emitted messages: Trader.outOfSyncWarning - The internal order pool and what's actually on the exchange may be out of sync Trader.trade-finalized - An order is complete (done) Trader.my-orders-cancelled - A call to cancel all orders in this orderbook has completed Trader.all-orders-cancelled - A call to cancel ALL of the user's orders (including those placed elsewhere) has been completed Trader.order-placed - Emitted after an order has been successfully placed Trader.order-cancelled - Emitted after an order has been cancelled Trader.trade-executed - emitted after a trade has been executed against my limit order Trader.place-order-failed - A REST order request returned with an error Trader.cancel-order-failed - A Cancel request returned with an error status
